# Глосарій мануальника

Польовий довідник різновидів manual QA — очима соцмереж.

Живе тут: [manual-qa-glossary.vercel.app](https://manual-qa-glossary.vercel.app) *(після деплою)*

---

## Як додати новий тип

### Структура терміну

Кожен термін — це об'єкт у масиві `TERMS` у файлі `index.html`:

```js
{
  term: "Назва типу",          // обов'язково: повна назва, з великої літери
  aka: "інша назва, slang",   // необов'язково: альтернативні назви через кому
  def: "Визначення.",          // обов'язково: опис у дусі соцмереж
  tags: ["тег1", "тег2"],     // обов'язково: 2–4 теги
  letter: "Н",                 // обов'язково: перша літера терміну (для групування)
}
```

### Правила

1. **Тон** — іронічний або нейтральний. Не образливий, не захисний. Це фольклор, не стандарт.
2. **Джерело** — термін має бути реальним: зустрічатися в LinkedIn, Twitter/X, Telegram-каналах про QA, Reddit, DOU тощо.
3. **Визначення** — 1–3 речення. Коротко і влучно. Можна цитату, якщо характерна.
4. **Теги** — 2–4 штуки, малими літерами, українською. Описують характеристику, а не оцінку.
5. **Унікальність** — перевір, чи немає вже схожого терміну. Синонім — не новий термін.
6. **Сортування** — масив `TERMS` сортується за алфавітом вручну. Додай термін на відповідне місце.

### Приклад

```js
{
  term: "Мануальщик-філософ",
  aka: "QA-думець, тестувальник без тест-кейсів",
  def: "Більше міркує про якість, ніж тестує. Веде нотатки про природу багів. На дейлі каже «а чи правильну річ ми взагалі будуємо?»",
  tags: ["рефлексія", "процес", "якість"],
  letter: "М",
},
```

### Чеклист перед PR

- [ ] Термін зустрічається в соцмережах (є хоч одне реальне джерело)
- [ ] Визначення не довше 3 речень
- [ ] 2–4 теги, українською
- [ ] `letter` відповідає першій літері `term`
- [ ] Термін вставлено в алфавітному порядку в масиві
